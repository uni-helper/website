`@uni-helper/unh` 提供了小程序开发者工具配置功能，可以在开发小程序时自动打开对应平台的开发者工具，提升开发效率。

## 基本配置

在 `unh.config.ts` 中配置开发者工具选项：

```ts [unh.config.ts]
import { defineConfig } from '@uni-helper/unh'

export default defineConfig({
  // 小程序开发者工具配置
  devtools: {
    // 是否自动打开小程序开发者工具
    open: true,
  },
})
```
::alert{type="success" icon="lucide:lightbulb"}
  若未配置 `cliPath`，默认会根据平台自动识别路径。
::
::alert{type="info" icon="lucide:link" to="https://github.com/uni-helper/unh/tree/main/src/libs/devtools/handlers" target="_blank"}
  目前仅支持自动打开微信开发者工具。如需打开其他平台的开发者工具，欢迎参与贡献。
::

## 配置选项

### `open`
::field{name="open" type="boolean" defaultValue="false"}
是否自动打开小程序开发者工具
::

### `cliPath`
::field{name="cliPath" type="Record<MPPlatform, CliPathConfig>"}
小程序开发者工具路径配置，默认会自动根据平台识别路径，若需自定义路径可以为不同平台指定不同的 CLI 路径
::

## 高级配置示例

```ts [unh.config.ts]
import { defineConfig } from '@uni-helper/unh'

export default defineConfig({
  devtools: {
    // 自动打开开发者工具
    open: true,
    // 自定义各平台开发者工具路径
    cliPath: {
      'mp-weixin': '/Applications/wechatwebdevtools.app/Contents/MacOS/cli',
    },
  },
})
```

## 注意事项

- 确保已安装对应平台的小程序开发者工具
- 路径配置需要指向正确的 CLI 工具位置
- 不同平台的开发者工具 CLI 路径可能不同，请根据实际情况配置
- Windows 系统下的路径格式与 macOS 不同，请使用适当的路径分隔符
